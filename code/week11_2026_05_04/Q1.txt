import jax.numpy as jnp
import pandas as pd
import numpy as np

# 1. 讀取資料 (假設檔案已上傳至 Colab 根目錄)
# 如果路徑不同，請修改檔案名稱
file_path = 'data_logmap.csv' 
raw_data = pd.read_csv(file_path, header=None).values

t = raw_data[:, 2]      # 標籤 (0 或 1)
X = raw_data[:, [0, 1]] # 特徵 (x0, x1)
n_samples = len(t)
sig2 = 10               # 正則化參數

# 2. 定義模型與損失函數 (MAP 估計)
def sigmoid(z):
    return 1 / (1 + jnp.exp(-z))

def loss_fn(w):
    # 計算預測值 y
    # 注意：這裡根據你的 w=[0,0] 設定，假設沒有額外加 bias 項
    y = sigmoid(jnp.dot(X, w))
    
    # 負對數概似函數 (Cross Entropy) + L2 Regularization (1/2*sig2 * w^2)
    # 小心 log(0) 的數值穩定性，JAX 通常處理得不錯
    cross_entropy = -jnp.sum(t * jnp.log(y + 1e-9) + (1 - t) * jnp.log(1 - y + 1e-9))
    regularization = (1 / (2 * sig2)) * jnp.sum(w**2)
    return cross_entropy + regularization

# 3. 準備牛頓法所需的梯度與 Hessian
grad_fn = jax.grad(loss_fn)
hess_fn = jax.jacfwd(jax.jacrev(loss_fn))

# 4. 開始迭代
w = jnp.array([0.0, 0.0]) # 初始權重
w_history = []

print(f"{'Iteration':<10} | {'w0':<12} | {'w1':<12}")
print("-" * 40)

for i in range(9): # 執行 9 次迭代
    g = grad_fn(w)
    H = hess_fn(w)
    
    # 牛頓更新公式: w = w - H^-1 * g
    w = w - jnp.linalg.inv(H) @ g
    w_history.append(w)
    
    print(f"{i+1:<10} | {w[0]:>12.6f} | {w[1]:>12.6f}")

# 5. 輸出第 9 次迭代的 w1 (四捨五入至小數點後四位)
final_w1 = round(float(w_history[8][1]), 4)
print("\n" + "="*40)
print(f"第 9 次迭代的 w1 為: {final_w1:.4f}")
print("="*40)

重點解析：Hessian 的物理意義：在邏輯回歸中，Hessian 矩陣 $H = \Phi^T R \Phi + \lambda I$。其中 $R$ 是一個對角矩陣，對角元為 $y_n(1-y_n)$，這代表了每個樣本點的「不確定性」權重。為什麼要用 jnp：在 Colab 中，使用 jax.numpy 可以讓 JAX 追蹤運算圖，這樣 jax.grad 才能正確算出偏微分。關於第 9 次迭代：程式碼中的迴圈執行 9 次，w_history[8] 即代表完成第 9 次更新後的結果。請注意 $w$ 的索引：w[0] 通常對應 $w_0$，w[1] 對應 $w_1$。如果你的 $x$ 矩陣第一欄是常數項（全為 1），那 $w_1$ 就會是斜率項。提示：如果執行時報錯找不到檔案，請確認 pd.read_csv 的路徑是否與你上傳至 Colab 的檔名完全一致。如果答案不對，請確認你的 $X$ 是否需要補上一欄 1 作為截距項（Intercept）。

### 1. 公式一致性檢查
*   **Newton's Method (牛頓法更新規則)**：
    你的公式 $x_{n+1} = x_n - \frac{f'(x)}{f''(x)}$ 是標準的單變數優化公式。在多維空間（向量 $\mathbf{w}$）中，它會演變成：
    $$\mathbf{w}_{new} = \mathbf{w}_{old} - \mathbf{H}^{-1} \nabla E(\mathbf{w})$$
    這正對應到你程式碼中的 `w = w - np.linalg.inv(Hessian) @ Jacobian`。

*   **Hessian 的計算方式**：
    你提到的 `jax.jacfwd(jax.jacrev(f))` 是 JAX 官方推薦計算 Hessian 的最優做法。`jacrev` 先算梯度（反向傳播），`jacfwd` 再對梯度算導數（正向傳播），這在處理 $\mathbb{R}^n \to \mathbb{R}$ 的標量函數時效率最高。

### 2. 實驗設定一致性 (Convergence)
*   **初始值與 $\sigma^2$**：
    設定 $\mathbf{w} = [0, 0]^T$ 且 $\sigma^2 = 10$（正則化強度），並在 **9 次迭代** 內收斂，這是一個非常經典的教學案例。這通常出現在分類問題中，證明牛頓法在損失函數是強凸（Strongly Convex）時具備**二階收斂速度**（比梯度下降快得多）。
*   **MAP 與 唯一性**：
    老師提到「因為唯一性，收斂值必為 MAP」，這是因為邏輯回歸加上 $L_2$ 正則化（由 $\sigma^2$ 提供）後的負對數概似函數是 **嚴格凸函數 (Strictly Convex)**，所以它只有一個全局最小值。

### 3. 缺點與限制 (Flaws) 說明
你列出的三點與老師的教學邏輯非常契合：
1.  **Not always global optimum**：雖然邏輯回歸是凸的，但在更複雜的模型（如神經網路）中，牛頓法可能會卡在鞍點或局部極小值。
2.  **Point estimate is not so Bayesian**：牛頓法找到的是一個「點」（MAP），真正的貝氏方法應該要考慮參數的整個**分布**（即計算 Posterior）。
3.  **Decision boundaries are parallel lines**：在線性邏輯回歸中，不論你怎麼疊代，決策邊界永遠是線性的。若要解決非線性問題，必須透過 Feature Mapping 或 Kernel 方法。
